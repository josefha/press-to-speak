# PressToSpeak macOS Brand Guidelines

## 1. Core Brand Colors

- `Background / Canvas`: `#F4EFE9`
- `Primary Ink (brand black)`: `#18181B`
- `On Ink (light text on dark surfaces)`: `#FFFFEB`

## 2. Supporting UI Colors

- `Card surface`: `#FFFDF7`
- `Soft surface`: `#F6EFE5`
- `Accent soft surface`: `#ECE6DD`
- `Border`: `#DDD3C7`
- `Strong border`: `#CEC1B4`
- `Muted text`: `#6D665D`

## 3. Semantic Status Colors

- `Success`: `#2E9C73`
- `Success soft`: `#E6F4ED`
- `Warning`: `#C98524`
- `Warning soft`: `#FBEFD9`
- `Error (softened)`: `#C56B70`
- `Error soft`: `#FBECEE`

## 4. Typography

- `UI font`: Geist
- `Brand/display font`: Outfit

Usage:
- Use Geist for controls, form text, utility labels, and body copy.
- Use Outfit for section headings and primary brand moments.

## 5. Button System

Two button styles only:

- `Primary`: dark fill (`#18181B`) with light text (`#FFFFEB`)
- `Secondary`: light card fill with ink text and defined border

Default sizing:
- `Min height`: `40pt` (matches Create Account / Log In sizing)
- `Corner radius`: `10pt`
- `Horizontal padding`: `14pt`

## 6. Usage Rules

- Prefer `#FFFFEB` over pure white when rendering text/icons on dark brand surfaces.
- Use semantic status colors consistently for badges, inline notices, and banners.
- Keep warning hue in the existing amber direction; avoid switching warning to red.
- Do not introduce one-off button variants unless a new product requirement explicitly needs it.
